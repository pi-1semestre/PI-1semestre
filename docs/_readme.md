# Documentação do Projeto ApoioPet

## Visão Geral

Este diretório `docs` agrega **todas as documentações geradas e utilizadas dentro do projeto ApoioPet**. Funciona como o repositório central de referência para toda a documentação do sistema, garantindo organização, rastreabilidade e fácil acesso a informações críticas do projeto.

## Estrutura de Pastas

A documentação está organizada em **3 pastas principais**, cada uma com responsabilidades específicas:

### 📁 Backend
Agregação de todas as documentações referentes ao **banco de dados** e **backend** do projeto.

**Conteúdo típico:**
- Diagramas de banco de dados (ER, DER)
- Documentação de APIs e endpoints
- Especificações técnicas de serviços
- Fluxos de dados
- Diagramas de arquitetura backend
- Documentação de integração e comunicação

### 📁 Frontend
Agregação de todas as documentações referentes ao **frontend** e **processo criativo** do projeto.

**Conteúdo típico:**
- Links de acessos para os protótipos
- Diagramas de fluxos de usuário (user flows)
- Design system
- Especificações de interface (UI/UX)
- Paletas de cores e tipografia

### 📁 Etc
Agregação de **documentações que fujam do escopo** das outras pastas.

**Conteúdo típico:**
- Casos de uso (use cases)
- Documentação de requisitos gerais
- Diagramas de arquitetura geral do projeto
- Planos de projeto
- Documentação de processos e fluxos de trabalho
- Outras documentações transversais
