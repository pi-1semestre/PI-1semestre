# Plano: Popular tabelas MySQL vazias após DROP/CREATE (Opção A)

**Objetivo**: Executar data.sql para popular as tabelas do banco `db_animal` após o usuário ter dropado/recriado o database.

**Problema Raiz Identificado**
- `spring.sql.init.mode=never` está ativo → Spring ignora completamente `data.sql`
- `ddl-auto=update` não recria tabelas após DROP
- Banco atualmente existe mas está vazio (0 tabelas, dados inexistem)

**Solução Escolhida**: Opção A - Alteração temporária em `application.properties`

---

## Passos do Plano

### 1. Diagnosticar estado atual (já concluído)
- [x] Verificado: tabelas não existem (`information_schema.tables` retornou 0)
- [x] Properties atuais: `ddl-auto=update`, `mode=never`

### 2. Alterar temporariamente `application.properties`

Editar `src/main/resources/application.properties`:

**Linha 11:**
```properties
spring.jpa.hibernate.ddl-auto=create
```
→ Mudar de "update" para "create"

**Linha 57:**
```properties
spring.sql.init.mode=always
```
→ Mudar de "never" para "always"

**Instrução crítica**: NÃO usar comentários inline na mesma linha (ex: `always # comment`). O formato `.properties` não suporta `#` como comentário inline.

### 3. Executar seeding

```bash
./mvnw spring-boot:run
```

Log esperado:
- Hibernate criará tabelas com `create table tb_administrador ...`, `tb_pelagem`, etc.
- DataSourceScriptDatabaseInitializer executará os INSERTs do data.sql
- Aplicação subirá com "Started DemoApplication"

### 4. Validação imediata no MySQL

```sql
USE db_animal;
SELECT 'tb_administrador', COUNT(*) FROM tb_administrador;  -- Esperado: 1
SELECT 'tb_pelagem', COUNT(*) FROM tb_pelagem;              -- Esperado: 12
SELECT 'tb_sexo', COUNT(*) FROM tb_sexo;                    -- Esperado: 2
SELECT 'tb_especie', COUNT(*) FROM tb_especie;              -- Esperado: 2
SELECT 'tb_porte', COUNT(*) FROM tb_porte;                  -- Esperado: 5
SELECT 'tb_animal', COUNT(*) FROM tb_animal;                -- Esperado: 4
SELECT 'tb_voluntarios', COUNT(*) FROM tb_voluntarios;       -- Esperado: 2
```

### 5. REVERTER imediatamente para estado seguro

Alterar `application.properties` de volta:

```properties
spring.jpa.hibernate.ddl-auto=update
spring.sql.init.mode=never
```

### 6. Verificar estabilidade

Reiniciar a aplicação 2-3 vezes:
```bash
./mvnw spring-boot:run
Ctrl+C
./mvnw spring-boot:run
```

Verificar que os dados persistem (mesmas contagens acima).

---

## Riscos e Mitigação

| Risco | Mitigação |
|-------|-----------|
| Comentário inline após valor (já causou erro `No enum constant`) | Garantir que `spring.sql.init.mode=always` fique sem `#` na mesma linha |
| `ddl-auto=create` apaga dados manuais futuros | Reverter para `update` imediatamente após seeding |
| Falha no data.sql por FK/order | data.sql está validado (ordem: admin→pelagem/sexo/especie/porte→emails→animais→mensagens→voluntarios) |
